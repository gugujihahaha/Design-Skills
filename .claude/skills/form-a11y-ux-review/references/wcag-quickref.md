# WCAG 2.1 快速参考 — 表单项相关准则

## 概述

本文档列出与 HTML 表单密切相关 WCAG 2.1 准则（A 级和 AA 级），作为 `form-a11y-ux-review` Skill 的可访问性维度参考。

---

## 原则 1：可感知 (Perceivable)

### 1.1.1 非文本内容 (A 级)
- 所有非文本内容（图标、CAPTCHA 图片）必须有等效的文本替代方案
- 表单项：装饰性图标使用 `aria-hidden="true"`，功能性图标必须有文字说明
- CAPTCHA：必须提供音频替代或 Honeypot 方案

### 1.3.1 信息和关系 (A 级)
- 通过呈现传达的信息、结构和关系必须以编程方式确定，或通过文本提供
- 表单项：使用 `<fieldset>` + `<legend>` 对 radio/checkbox 分组
- 表单项：使用 `<label>` 关联输入控件

### 1.4.1 颜色的使用 (A 级)
- 颜色不能作为传达信息的唯一视觉手段
- 表单项：错误状态不能只用红色边框表示，必须同时使用图标、文字标签等

### 1.4.3 对比度（最低）(AA 级)
- 文本与背景的对比度至少为 4.5:1（正常文本）或 3:1（≥18pt 或 ≥14pt 粗体文本）
- 表单项：label 文字、输入内容、错误消息均需满足此标准

### 1.4.11 非文本对比度 (AA 级)
- UI 组件（如输入框边框、焦点指示器）与相邻背景对比度至少 3:1
- 表单项：`:focus-visible` 轮廓的对比度 ≥3:1

---

## 原则 2：可操作 (Operable)

### 2.1.1 键盘 (A 级)
- 所有功能必须可通过键盘操作，无需特定的时间点要求
- 表单项：所有输入控件、按钮、自定义下拉框必须可通过 Tab 到达和操作

### 2.4.3 焦点顺序 (A 级)
- 焦点遍历顺序必须保持含义和操作的一致性
- 表单项：Tab 顺序应遵循 DOM 自然顺序，不使用 `tabindex > 0`

### 2.4.7 焦点可见 (AA 级)
- 任何键盘可操作的 UI 必须有可见的焦点指示器
- 表单项：`:focus-visible` 样式必须可见（≥3px，颜色对比 ≥3:1）

---

## 原则 3：可理解 (Understandable)

### 3.3.1 错误识别 (A 级)
- 如果输入错误被自动检测到，错误项必须被标识，并以文本向用户描述
- 表单项：使用 `aria-describedby` 将错误消息与对应输入框关联

### 3.3.2 标签或说明 (A 级)
- 当需要用户输入时，必须提供标签或说明
- 表单项：每个 `<input>`、`<select>`、`<textarea>` 必须有 `<label>` 或 `aria-label`
- 表单项：必填字段使用 `required` + `aria-required="true"`

### 3.3.3 错误建议 (AA 级)
- 如果检测到错误且有修正建议，应向用户提供建议
- 表单项：错误消息应包含如何修正的具体指引（如"密码至少8位，含字母和数字"）

### 3.3.4 错误预防（法律、财务、数据）(AA 级)
- 对于法律承诺或财务交易，提交必须可逆、检查或确认
- 表单项：重要操作前提供确认对话框

---

## 原则 4：健壮 (Robust)

### 4.1.2 名称、角色、值 (A 级)
- 所有 UI 组件的名称和角色必须以编程方式确定
- 表单项：native HTML 控件自带角色；自定义控件必须使用 ARIA 属性

### 4.1.3 状态消息 (AA 级)
- 状态消息不接收焦点时也能被辅助技术感知
- 表单项：错误/成功消息使用 `role="alert"` 或 `aria-live="polite"`

---

## 常见 WCAG 失败场景（表单）

| 场景 | 失败准则 | 严重程度 |
|------|----------|----------|
| `<input>` 没有 `<label>`，仅靠 placeholder | 1.1.1, 1.3.1, 3.3.2 | Critical |
| 错误消息仅在页面顶部，无 aria 关联 | 3.3.1, 4.1.3 | Critical |
| 仅用红色边框表示错误 | 1.4.1 | Critical |
| `outline: none` 且无自定义 focus 样式 | 2.4.7 | Critical |
| `tabindex` 乱序（1, 3, 2, 5, 4...） | 2.4.3 | Critical |
| 自定义下拉框不支持方向键 | 2.1.1 | Critical |
| 禁用按钮使用 `display: none` | 1.3.1 | Warning |
| placeholder 对比度 <3:1 | 1.4.3 | Warning |
